## 1. Demo 简介

### 是什么

FinPilot AI 是一个 AI 专家团队驱动的金融智能工作台，以 Web 应用形式交付，集成 AI 金融学习、AI 股票研究、AI 知识库管理三大核心能力，通过 Multi-Agent 协作系统让不同领域的 AI 专家像人类团队一样协同工作。

### 面向谁

- **金融专业学生**：备考 431 金融学综合，需要系统化学习路径和智能练习
- **431 金融专硕考研党**：需要针对性的知识点讲解和错题追踪
- **个人投资者**：需要快速获取股票分析、研报解读和投资建议

### 主要功能

**功能一：AI 金融学习助手**

覆盖 431 金融学综合全部考点，提供系统化的课程体系。每门课程包含知识点讲解、配套练习题、错题本管理。AI 根据用户答题情况动态调整学习路径，薄弱知识点自动强化。

![学习中心](docs/screenshots/learn.png)

**功能二：AI 股票研究员**

集成 AKShare 真实 A 股行情数据，支持股票搜索、K 线图分析、AI 深度研究报告生成。AI 基于 DeepSeek 大模型，涵盖基本面分析、财务数据解读、行业对比、投资建议，通过 SSE 流式实时输出。

![股票研究员](docs/screenshots/stocks.png)

**功能三：AI 知识库 RAG**

支持 PDF、Markdown 等文档上传，通过 ChromaDB 向量数据库进行语义存储。用户通过自然语言查询，AI 基于检索到的文档片段生成精准回答，并标注信息来源。完整的检索增强生成管线：文档上传 → 切块 → Embedding → 向量存储 → 语义检索 → 上下文组装 → LLM 生成 → 来源标注。

![知识库](docs/screenshots/knowledge.png)

**功能四：Multi-Agent 专家协作**

四位 AI 专家协同工作：学习专家负责知识讲解，研究分析师负责股票分析，知识专家负责文档问答，报告撰写专家负责结果整合。Orchestrator 调度中心自动识别用户意图、规划任务、分配专家、合并结果。

## 2. Demo 创作思路

### 灵感来源

灵感来自三个真实场景。第一个场景是备考 431 金融学综合时，教材内容庞杂，缺乏针对性的练习和反馈。第二个场景是日常投资中，想看一只股票的分析报告，需要在行情软件、研报平台、AI 工具之间反复切换。第三个场景是阅读大量研报后，想快速找到某个观点的出处，却要从几十份文档中逐一查找。这三个场景的共同点是：信息分散、工具割裂、AI 能力未被充分利用。

### 想解决的问题

- **痛点一：学习资源分散**。431 金融学综合涵盖货币银行学、公司理财、投资学三门课程，知识点超过 200 个。考生需要在教材、网课、真题之间反复切换，缺乏一体化的学习平台。
- **痛点二：研究工具割裂**。投资者分析一只股票，需要打开行情软件看 K 线、打开研报平台看分析、打开 AI 工具做总结。三个工具之间的信息无法打通，分析效率低下。
- **痛点三：知识管理困难**。研究人员积累的 PDF 研报、笔记文档越来越多，传统文件夹管理方式无法支持语义检索。想找到某个观点的出处，往往需要翻阅数十份文档。

### 为什么做这个方向

金融是 AI 落地的高价值场景。一方面，金融知识具有专业性强、时效性高的特点，通用 AI 难以给出精准回答；另一方面，金融领域的用户付费意愿强，商业化路径清晰。更重要的是，这个方向将 AI 的三种核心能力（对话、检索、分析）与金融的真实需求深度结合，而非简单的"套壳"应用。

选择这个方向还有一个个人原因：我亲历过 431 备考的全过程，这本身就是一个"我为自己开发"的产品，每个功能都是我真的需要的东西。这让它不只是一个参赛作品，更是一个真实的需求落地。

## 3. Demo 体验地址

### GitHub 仓库

**源码地址**：https://github.com/19823801199/FinPilot-AI

### 本地部署（完整体验推荐）

本地部署可以体验全部 AI 功能（对话、流式输出、Multi-Agent 协作、股票分析、知识库 RAG）。

**前置条件**：Node.js >= 20、Python >= 3.12、DeepSeek API Key

**配置 API Key**：

```bash
# 1. 克隆仓库
git clone https://github.com/19823801199/FinPilot-AI.git
cd FinPilot-AI

# 2. 复制环境变量模板并填入 DeepSeek API Key
cp .env.example .env
# 编辑 .env 文件，将 DEEPSEEK_API_KEY 替换为你的真实 Key
```

> AI 功能（对话、股票分析、知识库问答）依赖 DeepSeek API。你可以在 [platform.deepseek.com](https://platform.deepseek.com) 免费注册获取 API Key。

**启动后端**：

```bash
cd apps/api
python -m venv venv
venv\Scripts\activate        # Windows
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

**启动前端**：

```bash
cd apps/web
npm install
npm run dev
```

打开 http://localhost:3000 即可体验全部功能。

### Docker 一键部署

```bash
git clone https://github.com/19823801199/FinPilot-AI.git
cd FinPilot-AI
cp .env.example .env
# 编辑 .env，填入 DEEPSEEK_API_KEY
docker-compose up --build
```

## 4. TRAE 实践过程

### 开发完整流程

FinPilot AI 的开发分为 10 个阶段，全部在 TRAE 中完成：

**阶段1：需求分析**

在 TRAE 中通过多轮 AI 对话，将"做一个 AI 金融助手"的模糊想法转化为清晰的产品需求。AI 帮助识别了 3 类目标用户、3 大痛点，确定了 AI 学习、股票研究、知识库 RAG、Multi-Agent 协作 4 个核心功能模块，以及 MVP 范围（6 个页面、31 个 API 端点）。

**阶段2：产品设计**

基于需求分析，AI 辅助完成了 PRD 文档。核心产出包括：6 个页面的交互流程设计、Multi-Agent 协作的 8 步状态流转（意图识别→任务规划→任务拆分→分配专家→执行→收集结果→合并→最终回答）、以及 31 个 API 端点的接口定义。

**阶段3：UI 设计**

AI 根据"金融 AI 助手"的产品定位，直接生成了 TailwindCSS v4 的 `@theme` 全局样式配置。包含四级背景层次（#08090d → #111827 → #161922 → #1e293b）、三色强调系统（翡翠绿 #00e5a0、紫色 #6366f1、琥珀色 #f59e0b），以及 14 个基础 UI 组件规范。

**阶段4：数据库设计**

AI 根据 PRD 自动推导了 30 个数据模型——后端 6 个 Pydantic Schema 文件，前端 5 个 TypeScript 类型文件。核心设计包括泛型 `ApiResponse<T>` 统一响应格式、Agent 角色枚举、DAG 任务依赖字段。

**阶段5：工程初始化**

AI 帮助完成了 Monorepo 项目骨架（apps/web + apps/api + docker/）、TailwindCSS v4 配置、Docker 多阶段构建配置、环境变量模板（.env.example，11 个变量）。

**阶段6：前端开发**

AI 辅助完成了全部 6 个页面的前端开发：6 个页面、30+ 个 React 组件、18 个 Zustand Store、完整的 SSE 流式输出前端接收逻辑。核心难点是 SSE 连接的生命周期管理——AI 帮助设计了基于 AbortController 的方案，解决了组件卸载时连接泄漏问题。

**阶段7：后端开发**

AI 辅助实现了 6 个 API 路由模块、20 个业务服务类、完整的 AI 客户端封装（含指数退避重试）。31 个 API 端点全部实现，覆盖健康检查、AI 对话、股票分析、知识库管理、学习系统、Multi-Agent 协作。

**阶段8：RAG 开发**

AI 帮助实现了完整的检索增强生成管线：文档切块服务（500 字符/块，50 字符重叠）、ChromaDB 向量存储（支持 PersistentClient 和内存 Fallback 两种模式）、Embedding 服务、语义检索服务、RAG 主服务（检索 → 上下文组装 → LLM 生成 → 来源标注）。

**阶段9：Multi-Agent 开发**

AI 帮助实现了项目最具创新性的部分：4 位金融专家（学习导师、研究分析师、知识专家、报告撰写员）各配备独立的 system_prompt 和 task_prompt。Orchestrator 调度中心实现了意图识别 → 任务规划 → DAG 拓扑排序执行 → 结果合并的完整工作流。Prompt 工程在此阶段经历了多轮迭代，从"回答质量不高"到"可生成结构化金融分析报告"。

**阶段10：发布准备**

AI 辅助完成了代码审查（TODO/FIXME/console.log/debugger 清零、XSS 漏洞修复、未使用依赖清理、Mock 数据残留清理）和文档生成（README.md 20 章节、CHANGELOG v1.0.0、答辩材料 30 个 Q&A、演示脚本 3/5/8 分钟三版、最终发布报告）。
